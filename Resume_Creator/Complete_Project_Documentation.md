# AI Resume Creator — Complete Project Documentation

> Full engineering history from concept to Phase 6 completion.

---

## Project Overview
An AI-powered resume builder web application where users log in with Google, fill out structured forms about their background, upload supporting documents (PDFs, PPTs), and receive an elite, ATS-optimized resume generated by Google Gemini AI. The resume is rendered on an interactive A4 editor with rich text formatting, 3 template styles, and AI-powered re-analysis scoring.

---

## Phase 6: Advanced AI Resume Engine & Rich Editor

### 6.1 — Full 7-Output AI System Prompt
Deployed a comprehensive system prompt acting as:
- Expert resume writer (15+ years FAANG experience)
- ATS optimization specialist (targeting 90%+ scores)
- Technical recruiter with strict formatting rules

The AI now returns a unified JSON response containing:
1. **Resume Data** (structured JSON for the editor)
2. **ATS Score Analysis** (overall, keyword match, readability, formatting)
3. **Improvement Engine** (original → improved rewrites)
4. **Smart Suggestions** (actionable advice)

### 6.2 — Rich Text Formatting Toolbar
Built a sticky floating toolbar with:
- **Font Family**: Inter, Georgia, Arial, Courier New, Times New Roman, Verdana
- **Font Size**: Small → XL (5 levels)
- **Bold / Italic / Underline** buttons
- **Text Alignment**: Left, Center, Right
- **Text Color Picker**: Full spectrum color input
- **Clear Formatting**: Reset all formatting on selected text

### 6.3 — Resume Template Switcher
Three CSS-driven template styles:
- **Minimal**: Georgia serif, uppercase headings, clean separators
- **Modern**: Inter sans-serif, blue accents, bold headings (default)
- **FAANG**: Courier monospace, dense layout, ultra-concise spacing

### 6.4 — AI Re-Analysis System
- Created `/api/reanalyze` endpoint
- Extracts the current `innerText` from the edited resume
- Sends it back to Gemini for fresh ATS scoring including `visual_appeal`
- Updates the score dashboard, improvements, and suggestions in real-time

### 6.5 — ATS Score Dashboard
Visual panel showing:
- Overall ATS Score (0–100)
- Keyword Match percentage
- Readability score
- Formatting score
- Missing Keywords (red badges)
- Weak Bullet Points
- Sections to Improve

### 6.6 — Improvements Panel
Displays before/after comparisons:
- ❌ Original weak line (strikethrough)
- ✅ AI-rewritten stronger version

### 6.7 — Smart Suggestions Panel
Actionable advice cards for resume optimization

## Phase 7: Advanced Features (ATS Bar, Fix Resume, Job Match)

### 7.1 — Per-Template ATS Scoring
- Switching between Minimal/Modern/FAANG now auto-triggers re-analysis
- Template type is sent to the AI so it considers format-specific ATS implications
- Score bar label updates to show active template name

### 7.2 — Animated ATS Score Bar
- Dynamic progress bar (0-100) with smooth CSS transitions
- Color-coded: Red (0-50), Yellow (51-75), Green (76-100)
- Mini stats below: Keywords, Readability, Formatting scores

### 7.3 — Click-to-Add on Analysis Items
- **Missing Keywords**: Click to inject into Skills section (green highlight)
- **AI Improvements**: Click to replace original text with improved version
- **Smart Suggestions**: Click to append to resume as new bullet point
- **Sections to Improve**: Click to add suggestion text
- Hover reveals "+ ADD" badge on interactive items

### 7.4 — Fix My Resume Button
- One-click auto-rewrite using AI improvements
- Fetches fresh analysis, finds matching text, replaces with improved version
- Falls back to fuzzy matching if exact match fails
- Shows toast notification with count of applied fixes

### 7.5 — Apply for this Job (JD Matcher)
- Expandable panel with textarea for job description
- Sends resume text + JD to `/api/job-match`
- Returns: match score, missing keywords, already-present keywords
- Missing keywords are clickable (injected into Skills section)
- Match score displayed with color-coded value

### 7.6 — Toast Notification System
- Fixed-position bottom-right toast for all user actions
- Auto-dismisses after 3 seconds with slide-in/fade-out animation

## Phase 8: Word-Like Editor & More Templates

### 8.1 — Microsoft Word-Style Canvas
- Converted fragmented `contentEditable` spans to a unified A4 editable root
- Fully integrated `dangerouslySetInnerHTML` to allow raw structural edits
- Users can now click anywhere within the A4 boundary to type seamlessly

### 8.2 — Enhanced Formatting Toolbar
- Added **Strikethrough**, **Bulleted List**, **Numbered List**, **Horizontal Rule**
- Added **Indent**, **Outdent**, and **Full Justify**
- Added **Highlighter Color** (Text background color picker)
- Added **Undo** and **Redo** capabilities natively mapping to document commands

### 8.3 — Advanced Template Library
Included 4 new layouts (7 total):
- **Executive**: Garamond, double borders, formal line height
- **Creative**: Verdana, purple theme, side-bordered lists
- **Professional**: Calibri, navy blue header block backgrounds
- **Tech**: JetBrains Mono, dashed borders, code-aesthetic UI

## Phase 9: Tailwind CSS & Framer Motion UI Refactor

### 9.1 — Logic Fix: Template Re-Analysis
- Stopped auto re-triggering the Gemini `/api/reanalyze` call merely upon template change
- Ensures users don't face repeated loading latency while flipping styles, as the content remains identical

### 9.2 — Tailwind UI Migration
- Installed PostCSS and Tailwind CSS
- Refactored hardcoded CSS side-panels (ATS Dashboard, Job Match, AI Improvements) to atomic Tailwind utilities (`w-full max-w-[960px] bg-white/95 rounded-xl shadow-md p-5 backdrop-blur-md border border-gray-100`)

### 9.3 — Framer Motion Animations
- Animated the ATS Progress Bar filling with `transition={{ duration: 1, ease: "easeOut" }}`
- Wrapped floating panels in `AnimatePresence` for smooth layout entering/exiting
- Added interactive hover/tap scaling to quick-action keyword chips using `whileHover` and `whileTap`

## Bug Fix Log
| # | Bug | Root Cause | Fix |
|---|---|---|---|
| 1 | `client_id is required` | Missing `.env` Google OAuth credentials | Populated `.env` correctly |
| 2 | `Export default doesn't exist` | `pdf-parse` ESM incompatibility with Turbopack | Replaced with CommonJS `require()` |
| 3 | `DOMMatrix` ReferenceError | `pdf-parse` uses browser APIs on server | Switched to `pdfreader` + `officeparser` |
| 4 | Gemini `404 Not Found` | `gemini-1.5-flash` not available | Switched to `gemini-2.5-flash` |
| 5 | `fflate` SSR crash | `react-to-pdf` → `jspdf` uses Web Workers | Used `next/dynamic` with `ssr: false` |
| 6 | File upload overwriting | React `<input>` not resetting after selection | Added `e.target.value = ''` reset |
